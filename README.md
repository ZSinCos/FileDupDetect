# 音乐重复文件检测工具 - 规格说明书

## 1. 项目概述

- **项目名称**: MusicDupDetect
- **项目类型**: Windows桌面应用
- **核心功能**: 扫描音乐文件夹，识别内容相同的音频文件（通过哈希、元数据、音频指纹），并提供可视化界面让用户选择保留的文件
- **目标用户**: 音乐收藏者，需要整理音乐库的用户

## 2. UI/UX 规格

### 2.1 窗口结构

- **主窗口**: 单窗口应用，包含工具栏、文件列表、详情面板
- **窗口尺寸**: 默认 1200x800，最小 900x600
- **窗口控件**: 标准的窗口边框（最小化、最大化、关闭）

### 2.2 视觉设计

#### 颜色方案
- **主色**: #1E88E5 (蓝色)
- **次色**: #424242 (深灰)
- **背景色**: #FAFAFA (浅灰)
- **强调色**: #FF5722 (橙红，用于删除操作)
- **成功色**: #4CAF50 (绿色)
- **文字色**: #212121 (深色文字)
- **边框色**: #E0E0E0

#### 字体
- **主字体**: "Segoe UI", 12px
- **标题**: "Segoe UI", 16px, 粗体
- **小字体**: "Segoe UI", 10px

#### 间距
- **组件间距**: 16px
- **内边距**: 12px

### 2.3 组件设计

#### 顶部工具栏
- **扫描按钮**: 蓝色按钮，带文件夹图标
- **停止按钮**: 灰色按钮
- **进度条**: 显示扫描进度和当前状态
- **状态标签**: 显示文件数量、已找到重复组数

#### 左侧文件列表
- **分组显示**: 按重复组展示，每组显示为一个卡片
- **卡片内容**:
  - 组标题: "重复组 X (N个文件)"
  - 展开/折叠按钮
  - 文件列表: 文件名、大小、时长、格式
  - 勾选框用于选择保留的文件
- **选中状态**: 背景高亮 #E3F2FD

#### 右侧详情面板
- **元数据显示**:
  - 标题 (Title)
  - 艺术家 (Artist)
  - 专辑 (Album)
  - 时长 (Duration)
  - 比特率 (Bitrate)
  - 采样率 (Sample Rate)
  - 文件大小
  - 文件路径
- **播放按钮**: 播放选中文件的片段

#### 底部操作栏
- **全选保留**: 选中每组的第一个文件
- **删除选中**: 删除用户勾选的需要删除的文件
- **结果统计**: 显示总共找到多少重复文件

### 2.4 交互行为

- **悬停效果**: 按钮和列表项悬停时背景变浅
- **选中效果**: 点击文件后详情面板更新
- **拖拽支持**: 支持拖拽文件夹到窗口进行扫描
- **右键菜单**: 文件右键显示"打开所在文件夹"、"播放"、"删除"

## 3. 功能规格

### 3.1 核心功能

1. **文件夹选择**
   - 点击"选择文件夹"按钮打开文件夹选择对话框
   - 支持拖拽文件夹到窗口
   - 递归扫描子文件夹

2. **音频文件扫描**
   - 支持格式: MP3, FLAC, WAV, AAC, OGG, M4A, WMA
   - 扫描过程中显示进度
   - 支持取消扫描

3. **重复检测算法** (优先级从高到低)
   - **音频指纹**: 使用 chromaprint/acoustid 识别内容相同的音频
   - **文件哈希**: MD5/SHA256 快速判断完全相同的文件
   - **元数据比对**: 标题+艺术家+时长+比特率相近
   - **码率/采样率**: 作为辅助参考

4. **结果展示**
   - 按重复组展示
   - 显示文件详细信息
   - 支持排序 (按文件名、大小、时长)

5. **文件操作**
   - 选择保留的文件
   - 批量删除不需要的文件
   - 删除前确认对话框

6. **播放预览**
   - 播放选中音频的片段 (可选功能)

### 3.2 数据流

```
用户选择文件夹
    ↓
扫描音频文件 → 提取指纹/哈希/元数据
    ↓
分组对比 → 生成重复组列表
    ↓
UI展示重复组
    ↓
用户选择保留的文件
    ↓
执行删除操作
```

### 3.3 模块设计

- **AudioScanner**: 扫描文件夹，收集音频文件
- **AudioFingerprint**: 提取音频指纹 (使用 chromaprint)
- **FileHasher**: 计算文件哈希
- **MetadataExtractor**: 提取元数据 (使用 mutagen)
- **DuplicateFinder**: 对比分组，找出重复
- **MainWindow**: UI主界面

### 3.4 边界情况

- 空文件夹: 提示"未找到音频文件"
- 无重复: 提示"未找到重复文件"
- 权限问题: 跳过无法读取的文件，记录日志
- 损坏文件: 跳过无法解析的文件
- 大文件夹: 分批处理，避免内存溢出

## 4. 技术实现

- **框架**: Python + PySide6
- **音频指纹**: pyacoustid + chromaprint
- **元数据**: mutagen
- **哈希**: hashlib

## 5. 验收标准

1. ✓ 可以选择文件夹并递归扫描
2. ✓ 正确识别内容相同的音频文件 (指纹/哈希)
3. ✓ 界面清晰展示重复组和文件信息
4. ✓ 用户可以勾选要保留的文件
5. ✓ 可以删除选中的文件
6. ✓ 扫描过程有进度反馈
7. ✓ 支持常见的音频格式
